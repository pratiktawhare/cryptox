## Step 1.3 — Setup & Auth Backend

#### [NEW] `server/src/routes/setup.js`
```javascript
// GET /api/setup/status
// Returns { initialized: boolean }
// Checks if any user exists in the database

// POST /api/setup/initialize
// Body: { username, password, apiKey?, apiSecret?, budget, maxLeverage, riskTolerance, trackedCoins }
// Creates user + hashes password + encrypts API key + stores preferences
// Uses Mongoose models: User.create(), ApiKey.create(), UserPreferences.create()
// Returns JWT token (auto-login after setup)
```

#### [NEW] `server/src/routes/auth.js`
```javascript
// POST /api/auth/login
// Validate credentials → return JWT in HttpOnly cookie

// POST /api/auth/logout
// Clear JWT cookie

// GET /api/auth/me
// Return current user info (protected by auth middleware)
```

#### [NEW] `server/src/middleware/auth.js`
- Extract JWT from HttpOnly cookie named `cryptox_token`
- Verify + decode → attach `req.user = { id, username }`
- Return 401 with clear message on invalid/expired

#### [NEW] `server/src/utils/encryption.js`
- `hashPassword(plain)` → bcrypt with 12 salt rounds
- `verifyPassword(plain, hash)` → boolean
- `encryptData(plain, key)` → `{ encrypted, iv, authTag }`
- `decryptData(encrypted, iv, authTag, key)` → plaintext
- AES-256-GCM with crypto.randomBytes(16) IV per operation

#### [NEW] `server/src/app.js`
```javascript
// Express setup
// - CORS configured for Vite dev server (localhost:5173)
// - Helmet for security headers
// - Rate limiter: 100 req/15min for auth routes
// - JSON body parser
// - Cookie parser
// - Mount routes: /api/setup, /api/auth, /api/keys, /api/preferences, etc.
// - Socket.IO server init
// - Global error handler
```
